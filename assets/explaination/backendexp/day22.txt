    🔍 Full Explanation

    @RequestMapping("/api/pets")
    All endpoints in this controller sit under /api/pets. Each method's mapping adds on top of this.

    Principal principal
    javapublic ResponseEntity<Pet> addPet(@RequestBody Pet pet, Principal principal) {

    Principal is injected automatically by Spring Security
    principal.getName() returns the email of the currently logged-in user
    This is possible because JwtFilter set the authentication in SecurityContext on every request
    You never pass the email manually — Spring pulls it from the token


    GET /api/pets
    java@GetMapping
    public ResponseEntity<List<Pet>> getAllPets() {
        return ResponseEntity.ok(petService.getAllAvailablePets());
    }

    No Principal needed — any logged-in user can see available pets
    Returns a list of all pets where available = true
    ResponseEntity.ok() wraps the list with 200 OK


    GET /api/pets/{id}
    java@GetMapping("/{id}")
    public ResponseEntity<Pet> getPetById(@PathVariable Long id) {
        return ResponseEntity.ok(petService.getPetById(id));
    }

    @PathVariable — reads the id from the URL
    /api/pets/3 → id = 3
    Returns single pet or throws exception if not found


    POST /api/pets
    java@PostMapping
    public ResponseEntity<Pet> addPet(@RequestBody Pet pet, Principal principal) {
        return ResponseEntity.ok(petService.addPet(pet, principal.getName()));
    }

    @RequestBody — reads JSON from request body and converts to Pet object
    principal.getName() — gets the shelter's email from the JWT token
    Passes both to PetService.addPet() which links pet to the shelter
    Only shelter accounts should call this — you'll enforce role-based access on Day 39


    DELETE /api/pets/{id}
    java@DeleteMapping("/{id}")
    public ResponseEntity<Void> deletePet(@PathVariable Long id, Principal principal) {
        petService.deletePet(id, principal.getName());
        return ResponseEntity.noContent().build();
    }

    ResponseEntity<Void> — no body to return, just a status code
    ResponseEntity.noContent().build() — sends 204 No Content
    PetService.deletePet() handles the authorization check internally
